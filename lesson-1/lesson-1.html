What is a web application? Web page?

There is a large diversity of different kinds of data that can be sent between a client and server. But, when we start speaking about web applications, we're talking about an entire program that is essentially delivered to a client by a server. For example, google docs is a web application that allows someone to write a document. It is very similar to microsoft word or pages, but it is a program that delivered to you laptop when you type docs.google.com into your web browser's search bar.

An entire web application is pretty fully featured. To support all these features, the application probably has multiple web pages. For example, when you're using google docs, there is a page that allows you to edit a document and a page that allows you to browse documents you have already writen.

Because someone can only view one page a time in their web browser, when a client makes a request of a server, the server will (generally) only deliver one web page.

A single web page of an application can be thought of as being composed of 3 distinct parts.

interface
the code that describes the elements on the page that interacts with the user--text, buttons, dropdowns, etcetera --along with how they look.
logic
the code tied to a web page that describes the behaviors of the web page based on how the user interacts with it
data
the dynamic information of a web page that is filled in by either the server or the client

When creating web pages that compose a web application, the first part we will focus on is the interface. This is because--and we will see this in future lessons--that the interface of a web page forms the skeleton of the page, while details of the web page (how it looks, its logic) will be--in a way--tacked onto this skeleton.

HTML

Most web page files are written using a coding language called HTML (though there are few others). This stands for hypertext markup language. This coding language is standardized and is universally understood by nearly every web browser. When your web browser (the client) request a web page from a server, what is most often sent back over the internet is an HTML file. Your browser then reads and interprets this code in order to display the web page.

HTML uses text wrapped in tags to create the page's interface, these tags are essentially instructions for the browser. For example, if write:

About me

 <h3> About me </h3>

I've written instructions inside my HTML file that tell the browser to create a heading (this is what the h3 tag means) that reads "About me"

HTML is read sequentially, meaning instructions are read one after another. So if I write:

About me

My favorite color is blue! 

 
                <h3> About me </h3>

                <p> My favorite color is blue! </p>

I've written instructions inside my HTML file that tell the browser to create a heading that reads "About me" followed by a paragraph that contains the text "My favorite color is blue!"

A note about blank spaces

When coding, most coders refer to blank spaces in their code as "whitespace". This comes from a time when every text editor was a program with black text on a white background that looked like this: 

        <!DOCTYPE html> 

        <h3> About me </h3>

        <p> My favorite color is blue! </p>

This is not really accurate anymore because most text editors come in a large variety of colors and most coders use non-white backgrounds because it is easier on your eyes. Unfortunately, this term is embedded in much of the literature you'll read about coding in general, so we are kind of stuck with it :/

You may noticed already or not, in HTML the whitespace inbetween tags doesn't mean much. For example, here is alternative with a bunch of added whitespace in the HTML code. Notice how it doesn't change what the browser shows at all:

About me

My fav orite color is blue! 

 
<h3> About                           me </h3>
<p> My      fav orite color      
is          blue! </p>

All the whitespace really does is tell the browser when two words are separate. Though, you must be careful. If you put whitespace inside a tag itself, this will cause a syntax error and might lead to some unexpected behavior.

< h 3> About me

My fav orite color is blue! 

 
<    h 3> About                       me </h3>
<p> My      fav orite color      
is          blue! </p>

As you've already seen, to create an HTML page, you must always start your document with: 

 <!DOCTYPE html>

This is a formality and helps the browser identify what kind of file it is reading the same way a file extension does.

Important notes

  • Web pages are HTML files that are sent from servers to clients
  • Always include <!DOCTYPE html> at the top of your .html file
  • HTML elements are displayed sequentially
  • Whitespace inside HTML code doesn't always matter, but it can create a syntax error if you're not careful
